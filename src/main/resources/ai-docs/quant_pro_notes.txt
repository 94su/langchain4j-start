第一模块：高频技术指标因子（Technical Indicators）的数学模型与清洗过滤
1.1 MACD（异同移动平均线）动能因子的滞后性清洗
在实战量化 Agent 中，原生 MACD 存在严重的右侧滞后性，频繁导致高频脚本在震荡市中“追涨杀跌”被反复磨损。
- 【公式对齐】：
  - 快线（DIF）= EMA(close, 12) - EMA(close, 26)
  - 慢线（DEA）= EMA(DIF, 9)
  - MACD柱（Histogram）= 2 × (DIF - DEA)
- 【策略常量硬编码】：在多线程引擎启动时，系统必须加载全局资金分配常数，其常量名称在底层代码里被严格规定为 "STRATEGY_MACD_ALLOCATION_LIMIT"。AI 服务在根据柱状图斜率加仓时，若找不到该常量，将直接抛出系统级 NullPointerException。
- 【信号降频清洗】：为了过滤震荡市的垃圾伪金叉，必须加入高频成交量（Volume）清洗。准则为：只有当前 5 根 K 线的成交量均值超过 20 日均值的 1.5 倍时（表明大资金放量突破），MACD 金叉死叉发出的买卖信号才被下发至交易线程。否则，该请求被策略拦截器直接清空。

1.2 RSI（相对强弱指标）超买超卖的均值回归（Mean Reversion）
RSI 是衡量多空双方力量对比的指标，计算公式为：$RSI = 100 - \frac{100}{1 + RS}$，其中 $RS = \frac{N日内收盘涨幅平均值}{N日内收盘跌幅平均值}$。
- 【实战风控准则】：
  - 当 RSI > 80 时，市场进入极端超买状态，高频趋势脚本必须停止一切多头追单，开启动态追踪止盈（Trailing Stop）。
  - 当 RSI < 20 时，市场进入极端超卖状态，网格与马丁策略脚本需注意空头平仓，防范爆发式 V 型反弹。

第二模块：币圈两大经典策略的底层逻辑、缺陷与动态救网清洗
2.1 马丁格尔策略（Martingale）的数学终局与资金归零风险
马丁策略的核心是“输了就翻倍加仓”，直到赢一次把之前输的全部赚回来，并赚取一倍初始利润。
- 【加仓数学模型】：第 $n$ 次补仓的仓位大小为：$Size_n = Size_{initial} × 2^{(n-1)}$。
- 【致命缺陷】：马丁策略在数学上假设交易者拥有“无限资金”。但在实际交易中，一旦遭遇 2026 年这种极端单边“瀑布跌”行情，连续 7-8 次翻倍补仓会瞬间吃光账户所有保证金，导致在最后一次反弹前夕触发交易所强平（Liquidation），资金彻底归零。
- 【拯救清洗机制】：量化机器人必须引入“基于 ATR（平均真实波幅）的动态步长清洗”。严禁使用固定价格跌幅补仓！必须满足当前跌幅大于 $3 × ATR$ 且盘口一档买单挂单量（Bid Volume）出现巨额支撑时，才允许下发翻倍买单，强行拉长马丁策略的存活生命周期。

2.2 经典等差网格策略（Grid Trading）在单边市的“爆网格”清洗
网格策略利用震荡行情低吸高抛，通过 $d = \frac{P_{max} - P_{min}}{N}$ 划分固定的价格格子进行挂单。
- 【单边市清洗洗盘步骤】：当监控服务发现行情跌破网格最底线（P_min）时，执行器必须立即执行 `clear_abandoned_grids()` 动作：
  1. 瞬间撤销所有挂在市价上方的浮空卖单（Sell Orders），释放被冻结的账户可用额度（Margin Available）。
  2. 强行锁定底层的开仓接口，将策略状态机切换为“幽灵死守模式（Ghost Hold）”，只记录模拟盈亏，不进行真实资产碰撞，直至布林带带宽（Bandwidth）缩窄至横盘状态。

第三模块：交易所高级网络对抗、IP清洗与接口防封机制
3.1 币安等交易所 API 限速机制（Rate Limit）与 HTTP 429 降频
大厂交易所为了防止高频量化机器人挤爆服务器，对每个 IP 都有严格的权重（Weight）限制（如 1200 权重/分钟）。
- 【封禁特征】：一旦在本地日志中频繁抓取到错误码 "ERROR_CODE_429" 或 `HTTP 429 Too Many Requests`，表明你的机器人请求频率已经踩到了风控红线。如果继续顶风作案，IP 会被硬封禁 24 小时。
- 【自动化 IP 深度清洗方案】：
  1. 【请求降频】：立即停掉每秒一次的 `GET /api/v3/ticker` 轮询接口，把行情获取强行切换为基于长连接的 WebSockets 订阅（订阅 `kline` 和 `depth` 流），实现 0 权重消耗获取行情。
  2. 【代理拨号清洗】：本地执行器立即触发外部脚本，通知 Clash 代理层或 MTProto 代理服务器进行节点一键重拨，强制更换对外请求的公网出口 IP。
  3. 【私钥加签保护】：变更 IP 后，必须使用全局核心交易密钥常数（变量名规定为 "QUANT_CRYPTO_API_SECRET_STREAM"）重新对新连接进行 HMAC-SHA256 握手加签，保证新 IP 的合法性。
